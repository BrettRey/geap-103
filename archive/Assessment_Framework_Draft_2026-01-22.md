# GEAP 103 Assessment Framework (Draft)

**Course:** GEAP 103 Basic Computer Skills
**Version:** Draft 2026-01-22
**Status:** For review

---

## Grading

- Percentage grades (Humber standard)
- Pass threshold: 50%
- Language level: Exit target B1 (IELTS 4.5 / TOEFL PBT 487 / TOEFL iBT 56)
- Entry level: A2+ to low-B1

---

## Assessment Overview

| Assessment | Weight | Timing | Primary CLOs |
|------------|--------|--------|--------------|
| Process Logs | 25% | Ongoing (Weeks 2-13) | 1, 4 |
| Technical Communication Portfolio | 20% | Weeks 4, 8, 12 | 5, 6 |
| Digital Artifacts | 30% | Weeks 4, 10, 14 | 3, 7 |
| Collaboration & Version Control | 15% | Weeks 6-14 | 2, 8 |
| Final Reflection | 10% | Week 14 | All |

**Total: 100%**

## Assessment Details

### 1. Process Logs (25%)

**What:** Weekly documentation of AI-assisted work sessions.

**Format:** Structured log entries (template provided) including:
- Task goal (what you wanted to accomplish)
- Initial prompt/instructions given to AI
- AI output evaluation (what worked, what didn't)
- Revisions made and why
- Final outcome

**Submission:** 10 logs across semester (best 10 of 12 submitted)

**CLO Alignment:**
- CLO 1: Articulate task goals, revise prompts
- CLO 4: Evaluate AI outputs, document decisions

**Grading Criteria:**
| Level | Description | % Range |
|-------|-------------|---------|
| Excellent | Clear goals, systematic evaluation, substantive revisions with rationale | 85-100 |
| Proficient | Goals stated, evaluation present, some revision documented | 70-84 |
| Developing | Goals vague, evaluation superficial, minimal revision evidence | 50-69 |
| Insufficient | Missing elements, unclear process | 0-49 |

---

### 2. Technical Communication Portfolio (20%)

**What:** Collection of short technical communication pieces demonstrating ability to communicate about technology clearly.

**Pieces (3 required):**

1. **Help Request** (Week 4) - 5%
   - Describe a technical problem clearly enough for someone to help
   - Include: context, what you tried, specific question

2. **Troubleshooting Log** (Week 8) - 7%
   - Document solving a technical problem
   - Include: error/issue, diagnosis process, solution, verification

3. **Collaboration Note** (Week 12) - 8%
   - Explain a technical contribution to a teammate
   - Include: what you did, why, how it connects to shared work

**CLO Alignment:**
- CLO 5: Interpret technical instructions, troubleshoot
- CLO 6: Communicate technical information clearly

**Grading Criteria:**
| Level | Description | % Range |
|-------|-------------|---------|
| Excellent | Clear, complete, appropriate terminology, would achieve purpose | 85-100 |
| Proficient | Understandable, mostly complete, minor gaps | 70-84 |
| Developing | Partially clear, missing key information, terminology issues | 50-69 |
| Insufficient | Unclear, incomplete, would not achieve purpose | 0-49 |

---

### 3. Digital Artifacts (30%)

**What:** Three digital artifacts of increasing complexity demonstrating file management and creation skills.

**Artifacts:**

1. **Organised File System** (Week 4) - 8%
   - Demonstrate logical folder structure
   - Consistent naming conventions
   - Evidence of version awareness (manual or Git)

2. **Mid-Semester Project** (Week 10) - 10%
   - Academic/professional artifact (document, slides, spreadsheet, or visualization)
   - Created with AI assistance
   - Meets accessibility basics (headings, alt text where applicable)

3. **Final Project** (Week 14) - 12%
   - More complex artifact or revision/extension of mid-semester work
   - Demonstrates iteration and refinement
   - Meets college standards for the artifact type

**CLO Alignment:**
- CLO 3: Organise, store, retrieve digital files
- CLO 7: Create and revise digital artifacts to college standards

**Grading Criteria:**
| Level | Description | % Range |
|-------|-------------|---------|
| Excellent | Well-organised, functional, accessible, meets/exceeds standards | 85-100 |
| Proficient | Organised, functional, mostly accessible, meets standards | 70-84 |
| Developing | Some organisation, functional with issues, accessibility gaps | 50-69 |
| Insufficient | Disorganised, non-functional, or missing accessibility basics | 0-49 |

---

### 4. Collaboration & Version Control (15%)

**What:** Evidence of collaborative work using Git/GitHub workflows.

**Components:**

1. **Individual Git Practice** (Weeks 6-8) - 5%
   - Commit history showing regular saves
   - Meaningful commit messages
   - Evidence of version restoration or branching

2. **Collaborative Project Contribution** (Weeks 9-14) - 10%
   - Contributions visible in shared repository
   - Evidence of reviewing/responding to others' work
   - Conflict resolution (if applicable)

**CLO Alignment:**
- CLO 2: Use AI-assisted Git/GitHub workflows
- CLO 8: Collaborate using shared workflows

**Grading Criteria:**
| Level | Description | % Range |
|-------|-------------|---------|
| Excellent | Regular commits, clear messages, meaningful collaboration evidence | 85-100 |
| Proficient | Consistent commits, adequate messages, collaboration present | 70-84 |
| Developing | Irregular commits, vague messages, minimal collaboration | 50-69 |
| Insufficient | Missing or incoherent version history | 0-49 |

---

### 5. Final Reflection (10%)

**What:** Written reflection (300-500 words, A2 level) on learning journey.

**Prompts:**
- What can you do now that you couldn't do at the start?
- What was most challenging? How did you work through it?
- How will you use these skills in your degree program?

**CLO Alignment:** All (metacognitive synthesis)

**Grading Criteria:**
| Level | Description | % Range |
|-------|-------------|---------|
| Excellent | Specific examples, genuine insight, clear connection to future use | 85-100 |
| Proficient | Examples present, some reflection, future connection stated | 70-84 |
| Developing | General statements, limited reflection, vague future connection | 50-69 |
| Insufficient | Missing substance, no real reflection | 0-49 |

## Notes

- Assessment weights favour process (45%) over product (30%), with collaboration (15%) and metacognition (10%) completing the picture
- Early assessments (Weeks 2-6) pitched at A2+; later assessments scaffold toward B1
- Git/GitHub introduced Week 3, assessed from Week 6 (allows 3 weeks of practice)
- Collaborative work concentrated in second half when students have foundational skills
