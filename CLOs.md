# GEAP 103: Basic Computer Skills – Course Learning Outcomes

**Semester:** Foundation (B1)
**Revised:** 2026-03-23 (from 8 CLOs to 7; see revision notes below)

---

## Category structure

The CLOs are grouped into three categories: Core Content, Language and Communication, and Applied and Integrated. These are organizational labels, not a taxonomy. All seven CLOs involve language use, and all involve application; the categories distinguish the primary emphasis of each outcome. For GPSE 100, this structure is required by degree-breadth regulations, which call for visible disciplinary content alongside language development. We use the same structure for GEAP 103 for consistency across the two new courses.

---

## Core Content (Digital and AI Competence)

1. **Manage digital files and project versions**
   Manage digital files and project versions using effective practices, including naming conventions, folder structures, cloud storage, and version-control tools (e.g., Git/GitHub, cloud version history).

2. **Troubleshoot routine technical problems**
   Troubleshoot routine technical problems by interpreting instructions and error messages, resolving the problem or describing it clearly enough to get help.

3. **Formulate clear requests to guide AI tools**
   Formulate clear requests to guide AI tools toward intended outcomes, checking whether outputs meet task requirements and revising language as needed.

4. **Evaluate AI-generated text**
   Evaluate AI-generated text against clear criteria (accuracy, completeness, relevance, bias, accessibility, academic integrity).

---

## Language and Communication

5. **Communicate about technical work and decisions**
   Communicate about technical work and decisions in short academic and professional formats (e.g., decision logs, help requests, troubleshooting logs, bug reports), explaining what was done, changed, or rejected and why, using relevant terms accurately.

---

## Applied and Integrated

6. **Produce digital products**
   Produce digital products (e.g., presentations, data visualisations, automated workflows, apps, and multimedia content) to college standards, including basic accessibility features.

7. **Collaborate with peers on digital projects**
   Collaborate with peers on digital projects using shared practices, contributing changes, tracking updates, and negotiating revisions through written and spoken communication.

## Assessment Implications

The CLOs suggest assessment should focus on:

- **Process documentation** (CLOs 3, 4): How students formulate requests, evaluate outputs, document decisions
- **Technical communication** (CLOs 2, 5): Quality of help requests, troubleshooting descriptions, decision logs, bug reports
- **Artifact packages** (CLO 6): The communicative package (product + documentation + sharing), not just the product
- **Collaboration evidence** (CLOs 1, 7): File management, Git history, contribution patterns, negotiated revisions

**Not assessed:** Code quality (AI writes the code), technical sophistication (AI determines that)

**Key principle:** The language level is B1; the project ambition can be high. Assessment rewards students who stretch their ability to articulate what they want, not students who execute trivial tasks perfectly.
