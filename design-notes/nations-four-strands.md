## Paul Nation's Four Strands Framework

For balanced language learning, programs should distribute learning time approximately equally across four strands:

1. **Meaning-Focused Input** (~25%): Listening and reading for comprehension
2. **Meaning-Focused Output** (~25%): Speaking and writing to communicate
3. **Language-Focused Learning** (~25%): Deliberate study of vocabulary, grammar, discourse
4. **Fluency Development** (~25%): Building speed and automaticity in all skills

**Key Principle:** The balance applies **across the entire program**, not necessarily within each individual course.

## SPECIFIC GAPS GEAP 103 CAN ADDRESS

### **Gap 1: FLUENCY DEVELOPMENT (Biggest Deficit)**

**Current Situation:**
- ~7% of program time vs. 25% recommended
- Students produce a lot, but always for grades (accuracy-focused)
- Limited opportunities for high-volume, low-stakes practice
- Fluency mentioned in CLOs but not systematically built

**Why This Matters:**
- B1 students need to build automaticity in basic structures
- Constant assessment creates performance anxiety
- Speed and ease are as important as accuracy for functional communication
- Fluency builds confidence

**What GEAP 103 Can Provide:**

#### **1. Typing Fluency (Essential Digital Skill)**
- **High volume:** Hours of typing practice
- **Low stakes:** Not graded on English accuracy, graded on completion
- **Speed focus:** Timed exercises, WPM goals
- **Authentic context:** Writing commit messages, comments, prompts for AI
- **Language benefit:** Kinesthetic reinforcement of language patterns

#### **2. CLI Command Fluency**
- **Repeated practice:** Same commands executed hundreds of times
- **Automaticity:** Commands become automatic (like formulaic sequences)
- **Error tolerance:** Mistakes are learning opportunities, not grade penalties
- **Builds confidence:** Mastery through repetition

#### **3. Code Reading Fluency**
- **Volume:** Read hundreds of code examples throughout semester
- **Pattern recognition:** Repeated exposure builds automatic comprehension
- **No production pressure:** Just reading, not writing complex code
- **Technical register:** Builds receptive fluency in specialized English

#### **4. AI Conversation Fluency**
- **Natural dialogue:** Chat with ChatGPT/Copilot conversationally
- **High volume:** Dozens of exchanges per session
- **Low stakes:** AI doesn't judge language errors
- **Iterative:** Rephrase, clarify, ask follow-ups naturally
- **Builds spoken fluency:** Preparation for human interaction

#### **5. Debugging Fluency**
- **Repeated cycles:** Error → Read message → Fix → Test (dozens of times)
- **Problem-solving fluency:** Fast pattern recognition
- **Reading fluency:** Process error messages quickly
- **Resilience:** Normalize errors as part of process

**GEAP 103 Fluency Focus:** ~50-60% of course time

## PROPOSED GEAP 103 ROLE: "FLUENCY SPECIALIST"

### **Course Focus Distribution (3 hrs/week × 14 weeks = 42 hours)**

| Focus Area | % Time | Hours | Rationale |
|------------|--------|-------|-----------|
| **Fluency Development** | 50% | 21 hrs | Fill the biggest gap (7% → 25% with GEAP 103) |
| **Meaning-Focused Input (Technical)** | 30% | 12.5 hrs | Add technical register to predominantly academic input |
| **Language-Focused Learning** | 15% | 6.5 hrs | Technical vocabulary, command syntax, formulaic sequences |
| **Meaning-Focused Output** | 5% | 2 hrs | Minimal - already overserved (commit messages, comments) |

### **Why This Distribution?**

#### **1. Fluency Development (50%) - PRIMARY MISSION**
**Fills critical gap:** Brings program fluency from 7% to ~18% (still below ideal 25%, but much better)

**Activities:**
- **Typing practice:** Timed exercises, WPM goals, blind typing
- **CLI fluency drills:** Repeat git commands until automatic
- **Code reading volume:** 100+ code examples throughout semester
- **AI conversation practice:** High-volume, low-stakes dialogue
- **Debugging cycles:** Repeated error → fix → test loops

**Assessment:**
- Based on volume and completion, not accuracy
- Build automaticity through repetition
- Low-stakes environment reduces anxiety

#### **2. Meaning-Focused Input (30%) - SECONDARY MISSION**
**Adds technical register:** Complements existing academic input

**Activities:**
- **Read Git documentation:** Commands, workflows, concepts
- **Read error messages:** Interpret system feedback
- **Read tutorials:** Follow step-by-step instructions
- **Read code:** Understand examples and patterns
- **Read Stack Overflow/GitHub:** Professional community norms

**Benefits:**
- Authentic technical English
- Purposeful reading (to solve problems)
- High-frequency technical vocabulary in context

#### **3. Language-Focused Learning (15%) - TERTIARY**
**Complements existing language focus (23%)**

**Activities:**
- **Technical vocabulary:** Deliberate instruction (commit, branch, merge, debug, error, syntax, etc.)
- **Command syntax:** Formulaic sequences (git add ., git commit -m "message")
- **Error message interpretation:** Pattern recognition
- **Prompt formulas:** Effective ways to ask AI assistants

**Note:** NOT traditional grammar—this is vocabulary and formulaic sequences for technical contexts

#### **4. Meaning-Focused Output (5%) - MINIMAL**
**Already overserved (50% currently)**

**Activities students will produce:**
- Git commit messages (brief, formulaic)
- Code comments (minimal, explanatory)
- Prompts for AI assistants (conversational)
- Questions on forums (if applicable)

**Assessment:**
- Minimal weight on written output quality
- Focus on functionality, not linguistic accuracy
- Not graded like EAPI/EAPA writing assignments

## RECOMMENDATIONS

### **1. Design GEAP 103 as "Fluency Specialist + Digital Skills"**

**Core Identity:**
- **50% Fluency Development** (fill biggest gap)
- **30% Technical Input** (expand register)
- **15% Technical Vocabulary** (deliberate learning)
- **5% Output** (minimal, functional)

### **2. Explicit Positioning in Program**

**Tell students:**
- "This course is your fluency practice space"
- "Other courses emphasize accuracy and assessment"
- "Here, you'll build speed and confidence through volume"
- "Errors are expected and educational"

### **3. Assessment Philosophy**

**DO assess:**
- Completion and effort (did you do the work?)
- Volume (how much practice?)
- Progress (are you getting faster/more automatic?)

**DON'T assess:**
- English accuracy in commit messages/comments
- Perfect code (code quality is secondary)
- Speed of initial learning (everyone progresses differently)

**Grading model:**
- Satisfactory/Unsatisfactory (like EAPC 106)
- OR: Portfolio of completed projects (did they work?)
- NOT: Traditional accuracy-based grading

### **4. Coordinate with Other Courses**

**Communicate to EAPI/EAPA instructors:**
- "GEAP 103 provides fluency practice your students need"
- "Don't expect perfect technical English from GEAP 103"
- "Students may use Git for your course projects (if desired)"
- "AI literacy from GEAP 103 transfers to your courses"

### **5. Market to Students**

**Positioning:**
- "Learn the tools your degree program expects"
- "Build skills for academic and professional success"
- "Work with cutting-edge AI technology"
- "Practice without pressure"
- "Relevant to any field (not just computer science)"
