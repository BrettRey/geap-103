# GEAP 103 Basic Computer Skills -- Week-by-Week Schedule

**Semester:** Foundation Semester (B1), 14 weeks, 3 hours/week

This schedule tells you what happens each week: the topic, the new words you will learn, what you need to hand in, and what you will write in your Goal Document.

---

## Phase 1: Discovery -- "What's possible?" (Weeks 1 -- 4)

### Week 1: What Can a Computer Do for Me?

| | |
|---|---|
| **New words** | folder, file path, file extension, cloud sync, version |
| **What's due** | Nothing |
| **Goal Document** | Write 3 -- 5 things you wish your computer could do for you. They don't have to be realistic. |

---

### Week 2: Getting AI to Do What I Mean

| | |
|---|---|
| **New words** | prompt, generate, revise, accept/reject, verification |
| **What's due** | Decision Log entries begin (you will submit 8 entries total between now and Week 11) |
| **Goal Document** | Which of your wishes could you explain most clearly? Did you discover any new ideas today? How has your thinking changed? |

---

### Week 3: Never Losing Work Again

| | |
|---|---|
| **New words** | repository, commit, snapshot, restore, push |
| **What's due** | Nothing (Git is introduced this week -- you will practise it before anything is graded) |
| **Goal Document** | If you could make something with a computer, what kind of thing might it be? Describe what someone would see. |

---

### Week 4: Where Am I Now?

| | |
|---|---|
| **New words** | No new words -- review all 15 terms from Weeks 1 -- 3 |
| **What's due** | **File System artifact package** (7%) -- your organised folder structure with consistent naming. **Oral Micro-Defence 1** (triads, 3 criteria) -- includes in-class written diagnostic and peer feedback. |
| **Goal Document** | What have you learned in 4 weeks? What can you do now that you couldn't before? What interests you most? Instructor reads your Goal Document this week and gives you brief comments. Peer response this week too. |

---

## Phase 2: Building -- "What do I need to learn?" (Weeks 5 -- 10)

### Week 5: Making Something Others Can Read and Use

| | |
|---|---|
| **New words** | heading (style), alt text, screen reader, Accessibility Checker, contrast |
| **What's due** | Nothing new due, but keep writing Decision Log entries. |
| **Goal Document** | Think about who will use what you build. What would make it hard for them? What accessibility issues should you plan for? |

---

### Week 6: Making Sense of Numbers

| | |
|---|---|
| **New words** | cell, row, column, formula, chart |
| **What's due** | Git episodes begin (you need 3 episodes between now and Week 12). **Decision Log feedback round 1 returned** -- the instructor gives you feedback on your first entries. |
| **Goal Document** | Does your project involve data? What would you need to organise and show? |

---

### Week 7: Working Together Without Breaking Things

| | |
|---|---|
| **New words** | pull request (PR), issue, fork, comment, review |
| **What's due** | Nothing new due. Keep working on Decision Log entries and Git episodes. |
| **Goal Document** | What part of your project is hardest? What could someone else help with? Write a GitHub Issue describing one task a partner could do. |

---

### Week 8: When Things Go Wrong

| | |
|---|---|
| **New words** | error message, troubleshoot, debug, help request, log |
| **What's due** | **Oral Micro-Defence 2** (triads, 5 criteria) -- includes in-class written diagnostic and peer feedback. |
| **Goal Document** | What is the riskiest part of your project? What is most likely to not work? Write a help request for a problem you think might happen. Instructor reads your Goal Document this week and gives you brief comments. Peer response this week too. |

---

### Week 9: Building My Thing (Part 1)

| | |
|---|---|
| **New words** | slide, layout, speaker notes, caption, colour contrast |
| **What's due** | Nothing new due. Keep working on Decision Log entries and Git episodes. |
| **Goal Document** | Write a concrete task list: What specifically will you build today? What do you need to finish before you can start the next part? |

---

### Week 10: Building My Thing (Part 2)

| | |
|---|---|
| **New words** | No new words -- consolidation |
| **What's due** | **Mid-Semester Artifact Package** (8%) -- your working artifact with process documentation. **Decision Log feedback round 2 returned** -- the instructor gives you feedback on entries 3 -- 5. |
| **Goal Document** | Compare what you planned with what you built. Write 3 -- 4 sentences about the gap between your plan and your result. |

---

## Phase 3: Creation -- "Does it do what I wanted?" (Weeks 11 -- 14)

### Week 11: What Do We Want to Build Together?

| | |
|---|---|
| **New words** | No new words -- consolidation |
| **What's due** | **Oral Micro-Defence 3** (triads, 7 criteria) -- includes in-class written diagnostic and peer feedback. **All 8 Decision Log entries due** (30%) -- curated and self-assessed. |
| **Goal Document** | What is your role in the group? What are you responsible for this week? |

---

### Week 12: Is It Working?

| | |
|---|---|
| **New words** | No new words -- consolidation |
| **What's due** | **Git Episodes due** (10%) -- all 3 episodes (Restore, Error Recovery, Collaboration). |
| **Goal Document** | Compare what you wrote in Week 4 with where you are now. What changed? Why? Peer response this week (cross-group). |

---

### Week 13: Making It Better

| | |
|---|---|
| **New words** | No new words -- consolidation |
| **What's due** | **Oral Evidence-Indexed Reflection** (10%) -- a one-on-one conversation with the instructor (4 minutes). You choose one CLO to talk about, the instructor picks a second one. Have your portfolio open with evidence ready. |
| **Goal Document** | Read your Goal Document from the beginning. What did you learn about what you want? How did your goals change? What can you do now that you couldn't in Week 1? Instructor reads your Goal Document this week and gives you final comments. |

---

### Week 14: Did I Build What I Wanted?

| | |
|---|---|
| **New words** | No new words |
| **What's due** | **Final Artifact Package** (10%) -- your most ambitious artifact with process documentation and evidence of sharing. Informal sharing/gallery session in class (not graded). |
| **Goal Document** | Is this what you set out to make? What changed along the way? (This week is oral sharing, not a written entry.) |

---

## Assessment Summary

| Assessment | Weight | When |
|---|---|---|
| AI Decision Logs (8 entries) | 30% | Entries: Weeks 2 -- 11. All 8 due Week 11. |
| Oral Micro-Defences + Written Artifacts (3 rounds) | 25% | Week 4, Week 8, Week 11 |
| Artifact Packages (3 packages) | 25% | Week 4 (7%), Week 10 (8%), Week 14 (10%) |
| Git Episodes (3 episodes) | 10% | Episodes: Weeks 6 -- 12. All due Week 12. |
| Oral Evidence-Indexed Reflection | 10% | Week 13 |
| **Total** | **100%** | |
